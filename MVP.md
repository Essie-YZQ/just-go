MVP - Version 1

Goal

Build the first working version of Just Go as quickly as possible.

The purpose of Version 1 is to validate the product idea, not to build a complete travel platform.

The MVP should feel polished, but remain technically simple.

⸻

Primary User Journey

A user opens Just Go.

↓

They enter their travel preferences.

↓

They receive a personalized travel recommendation within seconds.

↓

They feel confident enough to start booking their trip.

⸻

Pages

1. Home

Purpose:

Introduce the product.

Content:

* Just Go
* Tagline
* Short product description
* Start Planning button

⸻

2. Trip Planner

Collect:

* Destination
* Departure city
* Arrival date
* Trip length
* Hotel budget
* Transportation preference
* Food preference
* Travel interests
* Travel pace
* Trusted Sources

Trusted Sources examples:

* General recommendations
* Xiaohongshu
* Reddit
* Google Reviews
* Local recommendations

⸻

3. AI Results

Display:

* Go / No-Go Recommendation
* Best Area to Stay
* Transportation Advice
* Restaurant Suggestions
* Things To Do
* Day-by-Day Itinerary
* Backup Plan
* Booking Checklist

For Version 1, all results may be generated using mock data.

⸻

4. Preferences

Allow users to save:

* Budget
* Favorite cuisines
* Travel pace
* Transportation preference
* Trusted Sources
* Interests

Use localStorage only.

No database.

⸻

Technical Scope

Version 1 should use:

* Next.js
* React
* TypeScript
* Tailwind CSS

No backend.

No authentication.

No payment.

No APIs.

No AI API yet.

No browser extension.

Everything should run locally.

⸻

Design Principles

The product should feel:

* Fast
* Simple
* Premium
* Minimal
* Modern

Avoid unnecessary complexity.

Every page should help users make travel decisions faster.

⸻

Out of Scope

Do NOT build:

* Login
* User accounts
* Booking integrations
* Google Maps API
* Expedia API
* Booking.com API
* Xiaohongshu integration
* Weather API
* Database
* Payment
* Browser Extension

These will be added in future versions.

⸻

Success Criteria

Version 1 is successful if a user can:

1. Open the website.
2. Fill in their preferences.
3. Receive a believable travel plan.
4. Feel they understand the future direction of the product.

Version 1 is NOT intended to be production-ready.

Its purpose is to validate the product concept and establish a clean technical foundation for future iterations.
